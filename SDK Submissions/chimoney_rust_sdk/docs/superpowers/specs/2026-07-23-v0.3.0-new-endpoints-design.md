# Chimoney Rust SDK v0.3.0 — New Endpoints Design

## Overview

Add 22 strongly typed endpoints across 3 high-value domains to the Chimoney Rust SDK.

**Current version:** 0.2.0
**Target version:** 0.3.0
**Scope:** New API endpoints only (no breaking changes)

## Goals

1. Add Agents domain (11 endpoints) — AI agent lifecycle management
2. Add Beneficiaries domain (3 endpoints) — beneficiary CRUD
3. Add Multicurrency Wallets domain (8 endpoints) — multi-currency wallet operations
4. All new endpoints use strongly typed request/response structs
5. Each domain committed atomically by its own subagent

## Prerequisites

Add `PATCH` HTTP method to `ChimoneyClient` — required by agent update endpoints.

## Domain 1: Agents (11 endpoints)

**New file:** `src/types/agent.rs`

| Endpoint | Method | Path | Request Type | Response Type |
|----------|--------|------|--------------|---------------|
| Create agent | POST | `/v0.2.4/agents/create` | `CreateAgentRequest` | `AgentResponse` |
| List agents | GET | `/v0.2.4/agents/list` | — | `AgentListResponse` |
| Get agent | GET | `/v0.2.4/agents/get` | `id` query | `AgentResponse` |
| Update agent | PATCH | `/v0.2.4/agents/update` | `UpdateAgentRequest` | `AgentResponse` |
| Update policies | PATCH | `/v0.2.4/agents/update-policies` | `UpdateAgentPoliciesRequest` | `AgentResponse` |
| Suspend agent | POST | `/v0.2.4/agents/suspend` | `AgentIdRequest` | `AgentResponse` |
| Reactivate agent | POST | `/v0.2.4/agents/reactivate` | `AgentIdRequest` | `AgentResponse` |
| Get API key | GET | `/v0.2.4/agents/api-key` | `agentId` query | `AgentApiKeyResponse` |
| Manage API key | POST | `/v0.2.4/agents/api-key` | `ManageAgentApiKeyRequest` | `AgentApiKeyResponse` |
| Get transactions | POST | `/v0.2.4/agents/transactions` | `AgentTransactionsRequest` | `AgentTransactionsResponse` |
| Fund agent | POST | `/v0.2.4/agents/fund` | `FundAgentRequest` | `AgentResponse` |

**Types to create:**
- `Agent` — id, name, description, status, wallet_id, passport, meta, created_at
- `CreateAgentRequest` — name, description, meta
- `UpdateAgentRequest` — agent_id, name, description, meta (all optional except agent_id)
- `UpdateAgentPoliciesRequest` — agent_id, limits, capabilities, regions
- `AgentIdRequest` — agent_id
- `AgentApiKeyResponse` — agent_id, api_key_prefix, created_at
- `ManageAgentApiKeyRequest` — agent_id, action (create/rotate/revoke/delete)
- `AgentTransactionsRequest` — agent_id, sub_account (optional)
- `AgentTransactionsResponse` — status, data (Vec of transactions)
- `AgentResponse` — status, data (Agent)
- `AgentListResponse` — status, data (Vec of Agent)
